title: Use Cases
---

<ul class="alpha-list">
    <li>
        <h2>Improving recruitment</h2>
        <ul>
            <li>A data standard to describe the persona or profile of a volunteer and a potential volunteer</li>
            <li>A data standard to describe a volunteering opportunity</li>
            <li>A means to match personas / profiles to opportunities (brokerage)</li>
        </ul>
    </li>
    <li>
        <h2>Incentivising retention</h2>
        <ul>
            <li>A data standard to describe a volunteer’s current and previous work in an organisation or across organisations</li>
        </ul>
    </li>
    <li>
        <h2>Increasing awareness and marketing</h2>
        <ul>
            <li>A data standard to describe a volunteering opportunity and the personal or social outcomes it has, portable between advertising systems</li>
        </ul>
    </li>
    <li>
        <h2>Making volunteer skill checks, background checks, and training more efficient</h2>
        <ul>
            <li>A portable data standard to describe a volunteer’s  current and previous work, training, and accreditations from previous organisations</li>
        </ul>
    </li>
    <li>
        <h2>Mobilising volunteers in national emergencies (and events)</h2>
        <ul>
            <li>A data standard for volunteers and opportunities that is interoperable between national-scale calls for volunteering and local-level recruitment and management</li>
        </ul>
    </li>
    <li>
        <h2>Building evidence and reports for use in management decisions</h2>
        <ul>
            <li>A data standard to describe a volunteer’s work in an organisation that is aggregable and suited to management needs</li>
            <li>A means to interpret aggregate data that suits the skills and time constraints of the sector</li>
        </ul>
    </li>
    <li>
        <h2>Reporting impact (and potential impact) to donors (and potential donors)</h2>
        <ul>
            <li>A data standard to describe an organisation’s work and the social benefit it provides</li>
            <li>Controlled, standardised procurement requirements for funders to use</li>
        </ul>
    </li>
    <li>
        <h2>Ensuring digital platforms all speak the same language, allowing information to move between systems</h2>
        <ul>
            <li>Ensuring there is consensus amongst digital tool providers on a data standard and widespread adoption afterward.</li>
        </ul>
    </li>
</ul>
